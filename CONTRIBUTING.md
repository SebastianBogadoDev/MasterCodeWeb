# Contributing to MasterCodeWeb

Thank you for your interest in contributing to MasterCodeWeb.

## Mission

MasterCodeWeb aims to build fast, secure, accessible and SEO-optimized digital solutions following professional software engineering practices.

---

## Development Principles

Every contribution should follow these principles:

- Clean and maintainable code
- Mobile First
- Responsive Design
- Accessibility (WCAG)
- SEO Best Practices
- Performance Optimization
- Security by Design
- Scalability
- Documentation First

---

## Commit Convention

Use semantic commits.

Examples:

```text
feat: add new landing page
fix: resolve contact form validation
docs: improve README
style: improve spacing
refactor: simplify CSS structure
perf: optimize image loading
seo: improve metadata
security: sanitize form inputs
```

---

## Pull Requests Checklist

Before submitting a Pull Request:

- Code reviewed
- Responsive tested
- SEO checked
- Accessibility verified
- No console errors
- Lighthouse score validated
- Documentation updated

---

## Branch Strategy

- main → Production
- develop → Development
- feature/* → New features
- hotfix/* → Critical fixes

---

## Code Style

- Use meaningful variable names.
- Keep functions small.
- Comment only when necessary.
- Avoid duplicated code.
- Follow consistent formatting.

---

Thank you for helping improve MasterCodeWeb.
