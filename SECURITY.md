# Security Policy

## 🛡️ Supported Versions

The following versions of MasterCodeWeb currently receive security updates.

| Version | Supported |
|----------|-----------|
| v1.x | ✅ Yes |
| v0.x | ⚠️ Limited |
| Older versions | ❌ No |

---

## 📢 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please include:

- Description of the issue
- Steps to reproduce
- Potential impact
- Screenshots (if applicable)
- Suggested fix (optional)

---

## 📧 Contact

**contact@mastercodeweb.com**

Website:

---

## 🔒 Responsible Disclosure

Please do not publicly disclose security vulnerabilities before they have been reviewed and resolved.

MasterCodeWeb appreciates responsible disclosure and will investigate all legitimate reports.

---

## ✅ Security Best Practices

MasterCodeWeb follows modern web security practices, including:

- HTTPS everywhere
- Input validation
- Output escaping
- Protection against XSS
- Protection against CSRF
- Secure HTTP headers
- Content Security Policy (CSP)
- Secure cookies
- Dependency updates
- Regular code reviews
